Feed Revive — Obsidian clipping

Bookmarked tweets are written to your vault via the Local REST API community plugin. In that plugin enable the Non-encrypted (HTTP) Server and copy its API key below.

Notes match Obsidian Web Clipper's format and are saved as folder/Post by @handle on X <tweetid>.md. Set the folder to your Web Clipper folder (e.g. raw) to keep them together.

Feed revive

Splices your saved posts into the home timeline data so X renders them natively — images, video and buttons all work. If your feed ever misbehaves after an X update, turn this off to fall back to simplified text-only cards.

Sorts your Likes page globally across your captured likes history (the revive pool): as you scroll, each batch X loads is refilled from the sorted list, so it's one continuous order — not a per-batch shuffle. Falls back to reordering just the current batch until enough history has been captured (~50 posts).
